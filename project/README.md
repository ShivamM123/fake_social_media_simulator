## Installation & Setup

### 1. Clone or Download the Project

```bash
cd /path/to/project
```

### 2. Database Setup

**IMPORTANT**: If you had an old version of the database, you MUST reset it first!

#### Option 1: Reset Database (Recommended for First Time)

This will delete all existing data and create a fresh database:

```bash
mysql -u root -p12345678 < src/main/resources/reset_database.sql
```

#### Option 2: Migrate Existing Database

If you want to keep existing data, run the migration script:

```bash
mysql -u root -p12345678 < src/main/resources/migrate.sql
```

#### Option 3: Manual Setup

1. Open MySQL:
```bash
mysql -u root -p12345678
```

2. Run the reset script:
```sql
source src/main/resources/reset_database.sql
```

**Note**: After setting up the database, the application will automatically create a default admin user on first run:
- Username: `admin`
- Password: `admin123`

#### Database Configuration

The database connection is configured in `src/main/java/com/fakesocial/util/DatabaseConnection.java`:

- **URL**: `jdbc:mysql://localhost:3306/fake_social_db`
- **Username**: `root`
- **Password**: `12345678`

**Note**: Update these credentials in `DatabaseConnection.java` if your MySQL setup differs.

### 3. Build the Project

```bash
mvn clean compile
```

### 4. Run the Application

```bash
mvn exec:java
```

Or use the provided script:

## Database Schema

### Users Table
- `id`: Primary key
- `username`: Unique username
- `email`: Unique email
- `password`: Hashed password
- `created_at`: Account creation timestamp
- `is_fake`: Boolean flag for fake/AI users
- `is_admin`: Boolean flag for admin users

### Posts Table
- `id`: Primary key
- `user_id`: Foreign key to users table
- `content`: Post content (text)
- `created_at`: Post creation timestamp
- `is_ai_generated`: Boolean flag for AI-generated posts

### Comments Table
- `id`: Primary key
- `post_id`: Foreign key to posts table
- `user_id`: Foreign key to users table
- `content`: Comment content (text)
- `created_at`: Comment creation timestamp
- `is_ai_generated`: Boolean flag for AI-generated comments

### Likes Table
- `id`: Primary key
- `post_id`: Foreign key to posts table
- `user_id`: Foreign key to users table
- `created_at`: Like timestamp
- Unique constraint on (post_id, user_id)

## Troubleshooting

### Database Connection Error
- **Problem**: Cannot connect to MySQL
- **Solution**: 
  - Ensure MySQL server is running
  - Verify database credentials in `DatabaseConnection.java`
  - Check if database `fake_social_db` exists
  - Run the schema.sql script to create tables

### Compilation Errors
- **Problem**: Maven build fails
- **Solution**:
  - Ensure Java 11+ is installed: `java -version`
  - Ensure Maven is installed: `mvn -version`
  - Clean and rebuild: `mvn clean compile`

### Login Issues
- **Problem**: Cannot login
- **Solution**:
  - Verify username and password are correct
  - Check if user exists in database
  - Try registering a new account
  - Use default admin account: `admin` / `admin123`

### AI Content Not Appearing
- **Problem**: No AI posts or comments
- **Solution**:
  - Wait a few seconds (AI content generates every 10-25 seconds)
  - Check console for error messages
  - Verify fake users exist in database
  - Refresh the feed manually
